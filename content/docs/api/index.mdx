The Propeller Manager exposes a REST API for managing distributed workloads. Download the [Postman collection](../postman_collection.json) to explore the API interactively.

## Core APIs

- **[Health](/docs/api/health/get)** - Health check endpoint
- **[Metrics](/docs/api/metrics/get)** - Prometheus metrics endpoint
- **[Proplets](/docs/api/proplets/get)** - Manage proplet workers
- **[Tasks](/docs/api/tasks/get)** - Create and manage tasks
- **[Jobs](/docs/api/jobs/get)** - Manage task execution jobs
- **[Workflows](/docs/api/workflows/post)** - Deploy multi-task workflows (DAGs)

## Federated Learning

The FL API enables federated machine learning across distributed proplets:

- **[POST /fl/experiments](/docs/api/fl/experiments/post)** - Configure and start an FL experiment
- **[GET /fl/task](/docs/api/fl/task/get)** - Get FL task details for a proplet
- **[POST /fl/update](/docs/api/fl/update/post)** - Submit training updates (JSON)
- **[POST /fl/update_cbor](/docs/api/fl/update_cbor/post)** - Submit training updates (CBOR)
- **[GET /fl/rounds/\{round_id\}/complete](/docs/api/fl/rounds/round_id/complete/get)** - Check round completion status

See [Federated Learning](/docs/federated-learning) for architecture and usage details.
