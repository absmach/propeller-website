This is a federated learning example. Proplets train models locally on their private data, and the coordinator aggregates updates using the FedAvg algorithm. The model never leaves the device — only weight updates are shared.

For a comprehensive guide on federated learning architecture, motivation, and the complete training lifecycle, see the [Federated Machine Learning](/docs/federated-learning) documentation.

## Start FL Services

Start the FL infrastructure services:

```bash
cd propeller/examples/fl-demo
docker compose up -d model-registry aggregator local-data-store local-registry
```

Verify services are running:

```bash
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

Your output should look like this:

```
fl-demo-local-registry     Up About an hour    0.0.0.0:5000->5000/tcp
fl-demo-aggregator         Up 2 hours          0.0.0.0:8085->8082/tcp
fl-demo-local-data-store   Up 2 hours          0.0.0.0:8083->8083/tcp
fl-demo-model-registry     Up 2 hours          0.0.0.0:8084->8081/tcp
```

## Check Initial Model

Verify the model registry has the initial model (version 0):

```bash
curl -s http://localhost:8084/models/0 | jq .
```

Your output should look like this:

```json
{
  "b": 0,
  "version": 0,
  "w": [0, 0, 0]
}
```

## Seed Training Data

Add training data for your proplet:

```bash
curl -s -X POST "http://localhost:8083/datasets/YOUR_PROPLET_CLIENT_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "schema": "linear_regression",
    "proplet_id": "YOUR_PROPLET_CLIENT_ID",
    "data": [
      {"x": [1.0, 2.0, 1.0], "y": 10.0},
      {"x": [2.0, 1.0, 3.0], "y": 15.0},
      {"x": [3.0, 2.0, 2.0], "y": 19.0},
      {"x": [1.0, 3.0, 1.0], "y": 15.0},
      {"x": [2.0, 2.0, 2.0], "y": 17.0}
    ],
    "size": 5
  }'
```

Your output should look like this:

```json
{
  "proplet_id": "YOUR_PROPLET_CLIENT_ID",
  "size": 5,
  "status": "stored"
}
```

## Build Wasm

Build the FL client to WASM:

```bash
cd examples/fl-demo/client-wasm
GOTOOLCHAIN=go1.25.5 GOOS=wasip2 GOARCH=wasm go build -o fl-client.wasm fl-client.go
```

## Create Task

Create a federated learning task:

```bash
curl -X POST "http://localhost:7070/tasks" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "fl-client-train",
    "kind": "federated",
    "env": {
      "ROUND_ID": "round-1",
      "MODEL_URI": "http://localhost:8084/models/0",
      "MODEL_REGISTRY_URL": "http://localhost:8084",
      "DATA_STORE_URL": "http://localhost:8083",
      "COORDINATOR_URL": "http://localhost:8086",
      "HYPERPARAMS": "{\"epochs\":1,\"lr\":0.01,\"batch_size\":5}"
    }
  }'
```

Your output should look like this:

```json
{
```

## Running with Image URL

Alternatively, you can reference the WASM from an OCI registry instead of uploading:

```bash
curl -X POST "http://localhost:7070/tasks" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "fl-client-train",
    "kind": "federated",
    "image_url": "ghcr.io/YOUR_USERNAME/fl-client-wasm:latest",
    "env": {
      "ROUND_ID": "round-1",
      "MODEL_URI": "http://localhost:8084/models/0",
      "MODEL_REGISTRY_URL": "http://localhost:8084",
      "DATA_STORE_URL": "http://localhost:8083",
      "COORDINATOR_URL": "http://localhost:8086",
      "HYPERPARAMS": "{\"epochs\":1,\"lr\":0.01,\"batch_size\":5}"
    }
  }'
```

The proxy will fetch the WASM from the registry when the task starts.
