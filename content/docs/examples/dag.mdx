This is a DAG (Directed Acyclic Graph) workflow example. It demonstrates how to create and execute workflows where tasks have dependencies on other tasks. The manager validates the DAG to prevent cycles and ensures tasks execute in proper order based on their dependencies.

## Build Wasm

Build the addition example that will be used across all workflow tasks:

```bash
cd propeller
make addition
```

Your output should look like this:

```bash
GOOS=js GOARCH=wasm tinygo build -buildmode=c-shared -o build/addition.wasm -target wasip1 examples/addition/addition.go
```

## Create Workflow

Create a workflow with three tasks in a DAG structure where task2 and task3 depend on task1:

```bash
curl -X POST "http://localhost:7070/workflows" \
-H "Content-Type: application/json" \
-d '{
  "tasks": [
    {
      "name": "add",
      "inputs": [10, 20]
    },
    {
      "name": "add",
      "inputs": [5, 15],
      "depends_on": ["task1-id"]
    },
    {
      "name": "add",
      "inputs": [100, 200],
      "depends_on": ["task1-id"]
    }
  ]
}'
```

## Upload Wasm

Upload the wasm file to each task in the workflow:

```bash
curl -X PUT "http://localhost:7070/tasks/a1b2c3d4-1234-5678-9abc-def012345678/upload" \
-F "file=@$(pwd)/build/addition.wasm"

curl -X PUT "http://localhost:7070/tasks/b2c3d4e5-2345-6789-abcd-ef0123456789/upload" \
```

## Start Workflow

Start the first task (root of the DAG). The manager will automatically schedule dependent tasks when their dependencies complete:

```bash
curl -X POST "http://localhost:7070/tasks/a1b2c3d4-1234-5678-9abc-def012345678/start"
```

Your output should look like this:

```json
{ "started": true }
```

## Conditional Execution

Tasks can be configured to run only when their dependencies succeed or fail using the `run_if` field:

```bash
curl -X POST "http://localhost:7070/workflows" \
-H "Content-Type: application/json" \
-d '{
  "tasks": [
    {
      "name": "add",
      "inputs": [10, 20]
    },
    {
      "name": "add",
      "inputs": [5, 15],
      "depends_on": ["task1-id"],
      "run_if": "success"
    },
    {
      "name": "add",
      "inputs": [0, 0],
      "depends_on": ["task1-id"],
      "run_if": "failure"
    }
  ]
}'
```

In this example:
- Task 2 only runs if Task 1 succeeds
- Task 3 only runs if Task 1 fails (error handling path)

Task states: `0` (pending), `1` (scheduled), `2` (running), `3` (completed), `4` (failed), `5` (skipped), `6` (interrupted).
