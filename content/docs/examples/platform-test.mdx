This is a simple example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:platform` interface. It validates that the proplet runtime correctly exposes platform information and attestation capabilities to WASM modules via the `elastic:platform/platform` interface. The module calls `get-platform-info` to retrieve the platform type, version, and attestation support, then calls `attestation` with a 32-byte report-data blob to obtain attestation evidence.

To run this example, we need to set the `PROPLET_EXTERNAL_WASM_RUNTIME` environment variable to `""` so that the proplet runtime uses the default WASM runtime. We also need to set the `PROPLET_HAL_ENABLED` environment variable to `true` to enable the HAL.

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Create Task

To create a task for this example, we need to build the example to wasm and then create a task with the wasm file.

```bash
cd propeller
make platform-test
```

Your output should look like this:

```bash
cd examples/platform-test && cargo build --release
    Finished `release` profile [optimized] target(s) in 0.01s
cp examples/platform-test/target/wasm32-wasip1/release/platform-test.wasm build/platform-test.wasm
```

Now we can create a task with the wasm file:

```bash
curl -X POST "http://localhost:7070/tasks" \
-H "Content-Type: application/json" \
-d '{"name": "platform-test"}'
```

Your output should look like this:

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
```

## Upload Wasm

Now we need to upload the wasm file:

```bash
curl -X PUT "http://localhost:7070/tasks/a1b2c3d4-e5f6-7890-abcd-ef1234567890/upload" \
-F "file=@$(pwd)/build/platform-test.wasm"
```

## Start Task

Now we can start the task:

```bash
curl -X POST "http://localhost:7070/tasks/a1b2c3d4-e5f6-7890-abcd-ef1234567890/start"
```

Your output should look like this:

```json
{ "started": true }
```
