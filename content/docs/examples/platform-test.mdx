This is a simple example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:tee-hal` interface. It validates that the proplet runtime correctly exposes platform information and attestation capabilities to WASM modules via the `elastic:tee-hal/platform` interface. The module calls `platform-info` to retrieve the platform type, version, and attestation support, then calls `attestation` with a 32-byte report-data blob to obtain attestation evidence.

## Prerequisites

- A running proplet with HAL enabled, or a TDX-capable machine running the CVM setup from `hal/ubuntu/qemu.sh`
- The `wasm-hal-runner` binary (built from `proplet`) for standalone testing outside the task pipeline

To enable HAL in the proplet, set the following environment variables:

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Build

```bash
cd propeller
make platform-test
```

Your output should look like this:

```bash
cd examples/platform-test && cargo build --release
    Finished `release` profile [optimized] target(s) in 0.01s
cp examples/platform-test/target/wasm32-wasip1/release/platform-test.wasm build/platform-test.wasm
```

## Running Standalone on a TDX CVM

To test against real hardware attestation, copy the WASM file and the `wasm-hal-runner` binary into a TDX CVM and run:

```bash
wasm-hal-runner platform-test.wasm
```

## Create Task

To run this example via the task pipeline, build the WASM and create a task:

```bash
curl -X POST "http://localhost:7070/tasks" \
-H "Content-Type: application/json" \
-d '{"name": "platform-test"}'
```
