This example demonstrates running a **WASM Component Model** binary on propeller. Unlike the module mode example which passes arguments via `--invoke` CLI flags, component mode uses the `function_name` field and typed `inputs` — propeller encodes the arguments using [WAVE (WebAssembly Value Encoding)](https://github.com/bytecodealliance/wasm-tools/tree/main/crates/wasm-wave) and calls the exported function by name.

This example requires the external wasmtime runtime. Set the `PROPLET_EXTERNAL_WASM_RUNTIME` environment variable before starting proplet:

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME="/usr/local/bin/wasmtime"
make start-proplet
```

The proplet logs should confirm the external runtime is in use:

```bash
INFO Using external Wasm runtime: /usr/local/bin/wasmtime
```

## Build the Component

You need the following tools installed:

- [Rust](https://rustup.rs/) with the `wasm32-wasip1` target
- [wasm-tools](https://github.com/bytecodealliance/wasm-tools) (v0.200 or later) to componentize the binary
- [cargo-component](https://github.com/bytecodealliance/cargo-component) or `wit-component` CLI

```bash
cd propeller/examples/add-component
cargo build --target wasm32-wasip1 --release
wasm-tools component new target/wasm32-wasip1/release/add_component.wasm -o add_component.wasm
```

Your output should look like this:

```bash
   Compiling add-component v0.1.0
    Finished release [optimized] target(s) in 2.3s
```

## Create Task

Create a task using the `function_name` field to specify the exported component function. The `inputs` array contains the typed arguments that will be WAVE-encoded and passed to the function:

```bash
curl -X POST "http://localhost:7070/tasks" -H "Content-Type: application/json" -d '{"name": "add-component", "function_name": "add", "inputs": ["1", "2"]}'
```

## How It Differs from Module Mode

| | Module mode | Component mode |
|---|---|---|
| Binary format | Core WASM module | WASM Component Model |
| Function invocation | `cli_args: ["--invoke", "add"]` + `inputs: [10, 20]` | `function_name: "add"` + `inputs: ["1", "2"]` |
| Argument encoding | Passed as CLI positional args | WAVE-encoded typed values |
| WIT interface | Not required | Required |
| Build tooling | tinygo / emcc | cargo-component / wasm-tools |

## Invoking Directly with wasmtime

You can also test the component binary directly without propeller:

```bash
wasmtime run --invoke "add(1, 2)" add_component.wasm
```

Your output should look like this:

```bash
3
```
