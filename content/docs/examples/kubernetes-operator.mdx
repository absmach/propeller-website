This is a Propeller Kubernetes Operator example. It provisions SuperMQ, deploys the operator, registers an external proplet, and runs a WebAssembly addition task.

## Source Code

The source code is available in the [propeller-k8s-operator](https://github.com/absmach/propeller-k8s-operator) repository.

## Prerequisites

You need the following tools installed before proceeding:

- **Go 1.24 or later** — required to build and run the operator from source.

  ```bash
  go version
  ```

  Your output should look like this:

  ```bash
  go version go1.25.7 linux/amd64
  ```

- **Docker** — runs the SuperMQ stack and proplet containers.

  ```bash
  docker version --format '{{.Server.Version}}'
  ```

  Your output should look like this:

  ```bash
  27.5.1
  ```

- **k3d** — creates a lightweight local Kubernetes cluster.

  ```bash
  k3d version
  ```

  Your output should look like this:

  ```bash
  k3d version v5.8.3
  k3s version v1.31.5-k3s1 (default)
  ```

- **kubectl** — manages Kubernetes resources.

  ```bash
  kubectl version --client --short
  ```

  Your output should look like this:

  ```bash
  Client Version: v1.32.1
  ```

- **make** — runs the operator build targets.

- **jq** — parses JSON from SuperMQ API responses during provisioning.

- **mosquitto-clients** — used to verify MQTT connectivity (`mosquitto_pub`, `mosquitto_sub`).

You also need:

- The **Propeller repository** cloned locally with binaries already built. This guide assumes it is at `~/Documents/propeller`. If you have not built it yet, follow the [Getting Started guide](/docs/getting-started) first.
- The **Propeller k8s operator repository** cloned locally. This guide assumes it is at `~/Documents/propeller-k8s-operator`.
  ```bash
  git clone https://github.com/absmach/propeller-k8s-operator.git ~/Documents/propeller-k8s-operator
  ```

## Step 5 — Provision SuperMQ Credentials

The operator and proplet need SuperMQ credentials—a domain, channel, and two client IDs with keys—to authenticate on MQTT. You provision these through the SuperMQ HTTP API exposed on port 80 by nginx.

The default admin credentials are set in `docker/.env`:

```bash
SMQ_USERS_ADMIN_EMAIL=admin@example.com
SMQ_USERS_ADMIN_PASSWORD=12345678
```

### 5.1 — Get an admin access token

```bash
TOKEN=$(curl -s -X POST http://localhost/users/tokens/issue \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"12345678"}' | jq -r '.access_token')
echo "Token: ${TOKEN:0:30}..."
```

Your output should look like this:

```bash
Token: eyJhbGciOiJFZERTQSIsImtpZCI6Im...
```

### 5.2 — Create a domain

```bash
DOMAIN=$(curl -s -X POST http://localhost/domains \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"propeller-test","route":"propeller-test","permission":"admin"}')
echo $DOMAIN | jq .
```

Your output should look like this:

```json
{
  "id": "3053156a-1994-4776-9e18-8c5d8883647c",
  "name": "propeller-test",
  "route": "propeller-test",
  "status": "enabled",
  "created_by": "70ae1ade-ed7a-4c6d-a508-4715efe9215d",
  "created_at": "2026-03-04T11:14:51.559482Z"
}
```

Store the domain ID:

```bash
DOMAIN_ID=$(echo $DOMAIN | jq -r '.id')
echo "Domain ID: $DOMAIN_ID"
```

Your output should look like this:



### Verify MQTT Connectivity

Confirm the manager credentials can publish on MQTT. The MQTT client ID must match the SuperMQ client ID:

```bash
mosquitto_pub -h localhost -p 1883 \
  -i "$MGR_ID" -u "$MGR_ID" -P "$MGR_KEY" \
  -t "m/${DOMAIN_ID}/c/${CHAN_ID}/test" \
  -m '{"test":"hello"}' && echo "MQTT OK"
```

Your output should look like this:

```bash
MQTT OK
```

Confirm the alive payload the proplet sends. Subscribe to the liveness topic and capture one message:

```bash
timeout 15 mosquitto_sub -h localhost -p 1883 \
  -i "listener-$$" -u "$PROP_ID" -P "$PROP_KEY" \
  -t "m/${DOMAIN_ID}/c/${CHAN_ID}/control/proplet/alive" \
  -C 1
```

Your output should look like this:

```json
{"proplet_id":"b64502f0-25bc-4926-8fbc-ad508f75d96c","status":"alive","namespace":"default"}
```

The `proplet_id` field in alive messages is the proplet's `client_id`. The operator matches this value against the `spec.connectionConfig.clientId` field in `Proplet` CRs to identify which Proplet a message belongs to.

## Step 7 — Patch the Operator MQTT Topic Base

The operator's MQTT base topic must match SuperMQ's routing scheme. Open `internal/controller/proplet_controller.go` in the operator repository and verify line 45:

```go
var superMQBaseTopic = "m/%s/c/%s"
```

SuperMQ routes MQTT messages directly on the topic path `m/{domain_id}/c/{channel_id}/{sub-topic}`. The operator subscribes to `m/{domain_id}/c/{channel_id}/#` (wildcard), which matches all sub-topics. If this variable reads `"m/%s/c/%s/messages"` instead, the operator subscribes to the wrong path and never receives proplet messages.

The k8s operator will act as the manager. Stop the Docker manager container so it does not compete for the same MQTT subscriptions:

## Step 9 — Run the Operator

Run the operator locally against the k3d cluster. It connects to the same SuperMQ MQTT broker, using the manager credentials:

```bash
cd ~/Documents/propeller-k8s-operator

nohup go run ./cmd/main.go \
  --mqtt-address="tcp://localhost:1883" \
  --domain-id="3053156a-1994-4776-9e18-8c5d8883647c" \
  --channel-id="c19ded40-eeec-448e-86e9-42f490c766a4" \
  --client-id="0a96e62c-da03-45eb-9699-b7d8dfa843c9" \
  --client-key="8bd64edf-6cec-4cd8-aebd-a8e9695c8d90" \
  --mqtt-qos=2 \
  --metrics-secure=false \
  --health-probe-bind-address=":8082" \
```

## Step 11 — Run a Task

Create a Task that targets the docker proplet. This uses the addition WASM module (the same binary used in the [addition example](/docs/examples/addition))—a minimal module that exports a `main` function taking two integers and returning their sum:

```bash

apiVersion: propeller.propeller.abstractmachines.fr/v1
kind: Task
metadata:
  name: addition-wasm

  name: addition-wasm
  functionName: main
  file: "AGFzbQEAAAABBwFgAn9/AX8DAgEABwgBBG1haW4AAAoJAQcAIAAgAWoL"

    - 10

  propletSelector:
```

## Custom Resource Reference

### Proplet Spec

| Field                          | Type                 | Required       | Description                                          |
|--------------------------------|----------------------|----------------|------------------------------------------------------|
| `type`                         | `k8s` \| `external`  | Yes            | Where the worker runs                                |
| `k8s.image`                    | string               | Yes (k8s only) | Container image for the proplet pod                  |
| `k8s.logLevel`                 | string               | No             | Log verbosity: `debug`, `info`, `warn`, `error`      |
| `k8s.replicas`                 | int                  | No             | Number of pod replicas (default: 1)                  |
| `external.deviceType`          | string               | No             | Device label for scheduling (e.g. `raspberry-pi-4`)  |
| `external.capabilities`        | []string             | No             | Capabilities advertised for task matching            |
| `resources.cpu`                | string               | No             | CPU request/limit in Kubernetes format (e.g. `100m`) |
| `resources.memory`             | string               | No             | Memory request/limit (e.g. `256Mi`)                  |
| `connectionConfig.mqttAddress` | string               | Yes            | Broker URL (e.g. `tcp://mqtt:1883`)                  |
| `connectionConfig.domainId`    | string               | Yes            | SuperMQ domain ID                                    |
| `connectionConfig.channelId`   | string               | Yes            | SuperMQ channel ID                                   |
| `connectionConfig.clientId`    | string               | Yes            | SuperMQ client credential ID                         |
| `connectionConfig.clientKey`   | string               | Yes            | SuperMQ client secret                                |
| `connectionConfig.mqttQos`     | int                  | No             | MQTT QoS level 0–2 (default: 1)                      |
| `connectionConfig.mqttTimeout` | duration             | No             | Broker operation timeout (default: `10s`)            |

### Proplet Status

| Field                         | Description                                |
|-------------------------------|--------------------------------------------|
| `phase`                       | `Initializing`, `Running`, or `Offline`    |
| `conditions[Ready]`           | True when proplet is ready to accept tasks |
| `conditions[Connected]`       | True when MQTT heartbeat is current        |
| `lastSeen`                    | Timestamp of most recent alive heartbeat   |
| `taskCount`                   | Total tasks executed since registration    |
| `k8sStatus.readyReplicas`     | Ready pod replicas (k8s type only)         |
| `k8sStatus.availableReplicas` | Available pod replicas (k8s type only)     |

### Task Spec

| Field                              | Type              | Required | Description                                      |
|------------------------------------|-------------------|----------|--------------------------------------------------|
| `name`                             | string            | Yes      | Task name                                        |
| `functionName`                     | string            | Yes      | Exported WASM function to call                   |
| `file`                             | string            | No       | Base64-encoded WASM binary                       |
| `imageUrl`                         | string            | No       | OCI image reference for the WASM module          |
| `cliArgs`                          | []string          | No       | Arguments passed to the WASM runtime             |
| `inputs`                           | []int             | No       | Numeric inputs passed to the function            |
| `env`                              | map[string]string | No       | Environment variables available in the module    |
| `propletSelector.propletId`        | string            | No       | Target a specific Proplet by name                |
| `propletSelector.matchLabels`      | map               | No       | Target Proplets matching all labels              |
| `propletSelector.matchDeviceTypes` | []string          | No       | Target Proplets with matching device types       |
| `propletSelector.matchCapabilities`| []string          | No       | Target Proplets with all listed capabilities     |
| `daemon`                           | bool              | No       | If true, task runs indefinitely (default: false) |
| `resourceRequirements.cpu`         | string            | No       | Minimum CPU required                             |
| `resourceRequirements.memory`      | string            | No       | Minimum memory required                          |

### Task Status

| Field                   | Description                                                     |
|-------------------------|-----------------------------------------------------------------|
| `phase`                 | `pending`, `running`, `completed`, `failed`, or `interrupted`   |
| `assignedProplet`       | Name of the Proplet executing this task                         |
| `startedAt`             | When execution began                                            |
| `finishedAt`            | When execution ended                                            |
| `results`               | Return value from the WASM function                             |
| `error`                 | Error message when `phase` is `failed`                          |
| `conditions[Started]`   | True when the task began executing                              |
| `conditions[Completed]` | True when the task finished successfully                        |
