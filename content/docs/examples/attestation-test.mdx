A WASM component example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:hal@0.1.0` interface. It queries platform info, generates a random nonce, and requests TEE attestation evidence via the `hal-consumer` world.

## Prerequisites

- A running proplet with HAL enabled, or a TDX-capable machine.

To enable HAL in the proplet:

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Build

```bash
cd examples/attestation-test && cargo build --release --target wasm32-wasip2
```

Your output should look like this:

```bash
   Compiling attestation-test v0.1.0
    Finished `release` profile [optimized] target(s) in 2.34s
```

The component is at `target/wasm32-wasip2/release/attestation_test.wasm`.

## Start Task

```bash
curl -X POST "http://localhost:7070/tasks/b2c3d4e5-f6a7-8901-bcde-f12345678901/start"
```

Your output should look like this:

```json
{ "started": true }
```

On a TDX CVM, the proplet logs will show:

```bash
platform: IntelTdx v0.1.0
attestation: ok (evidence len=863)
```
