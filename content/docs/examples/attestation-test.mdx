This is a simple example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:tee-hal` interface. It validates that the proplet runtime correctly provides hardware-rooted attestation to WASM modules via the `elastic:tee-hal/platform` interface. The module calls `attestation` with a 16-byte nonce to request an attestation report from the underlying platform.

## Prerequisites

- A running proplet with HAL enabled, or a TDX-capable machine running the CVM setup from `hal/ubuntu/qemu.sh`
- The `wasm-hal-runner` binary (built from `proplet`) for standalone testing outside the task pipeline

To enable HAL in the proplet, set the following environment variables:

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Build

```bash
cd propeller
make attestation-test
```

Your output should look like this:

```bash
cd examples/attestation-test && cargo build --release
    Finished `release` profile [optimized] target(s) in 0.01s
cp examples/attestation-test/target/wasm32-wasip1/release/attestation-test.wasm build/attestation-test.wasm
```


## Create Task

To run this example via the task pipeline, build the WASM and create a task:

```bash
curl -X POST "http://localhost:7070/tasks" \
-H "Content-Type: application/json" \
-d '{"name": "attestation-test"}'
```

Your output should look like this:
