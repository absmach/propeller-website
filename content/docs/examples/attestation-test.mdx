A WASM core module example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:tee-hal` interface. It queries platform info, generates a random nonce, and requests TEE attestation evidence.

## Prerequisites

- A running proplet with HAL enabled, or a TDX-capable machine.

To enable HAL in the proplet:

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Build

```bash
cd examples/attestation-test && cargo build --release
```

Your output should look like this:

```bash
   Compiling attestation-test v0.4.0
    Finished `release` profile [optimized] target(s) in 2.34s
```

The module is at `target/wasm32-wasip1/release/attestation-test.wasm`.

## Start Task

```bash
curl -X POST "http://localhost:7070/tasks/b2c3d4e5-f6a7-8901-bcde-f12345678901/start"
```

Your output should look like this:

```json
{ "started": true }
```

The proplet logs will vary depending on the platform.

On a TDX CVM:

```bash
platform-info: {"platform_type":"IntelTdx","version":"0.1.0","attestation_support":true}
attestation: ok (evidence len=863)
evidence: {"measurements":{"mrtd":"...","rtmr0":"...",...},"platform":"intel-tdx","tdx_module_version":"1.5.0","version":"0.1.0"}
```

On a machine with no TEE (stub mode):

```bash
platform-info: {"platform_type":"None","version":"0.0.0","attestation_support":false}
attestation: ok (evidence len=2)
evidence: {}
```

The stub returns `{}` as the evidence. This is the fallback when `PROPLET_HAL_ENABLED=true` but no TEE hardware is present.
