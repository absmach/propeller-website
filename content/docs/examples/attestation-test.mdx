This is a simple example using the [Elastic HAL](https://github.com/elasticproject-eu/wasmhal) `elastic:attestation` interface, introduced in [PR #4](https://github.com/elasticproject-eu/wasmhal/pull/4). It validates that the proplet runtime correctly provides hardware-rooted attestation to WASM modules via the `elastic:attestation/hal` interface. The module calls `get-attestation` with a 16-byte nonce to request an attestation report from the underlying platform.

To run this example, we need to set the `PROPLET_EXTERNAL_WASM_RUNTIME` environment variable to `""` so that the proplet runtime uses the default WASM runtime. We also need to set the `PROPLET_HAL_ENABLED` environment variable to `true` to enable the HAL.

```bash
export PROPLET_EXTERNAL_WASM_RUNTIME=""
export PROPLET_HAL_ENABLED=true
```

## Create Task

To create a task for this example, we need to build the example to wasm and then create a task with the wasm file.

```bash
cd propeller
make attestation-test
```

Your output should look like this:

```bash
cd examples/attestation-test && cargo build --release
    Finished `release` profile [optimized] target(s) in 0.01s
cp examples/attestation-test/target/wasm32-wasip1/release/attestation-test.wasm build/attestation-test.wasm
```

Now we can create a task with the wasm file:

```bash
curl -X POST "http://localhost:7070/tasks" \
-H "Content-Type: application/json" \
-d '{"name": "attestation-test"}'
```

Your output should look like this:

```json
{
  "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
```

## Upload Wasm

Now we need to upload the wasm file:

```bash
curl -X PUT "http://localhost:7070/tasks/b2c3d4e5-f6a7-8901-bcde-f12345678901/upload" \
-F "file=@$(pwd)/build/attestation-test.wasm"
```

## Start Task

Now we can start the task:

```bash
curl -X POST "http://localhost:7070/tasks/b2c3d4e5-f6a7-8901-bcde-f12345678901/start"
```

Your output should look like this:

```json
{ "started": true }
```

## Invoking Using WasmTime
