This guide walks you through installing Propeller and running your first WASM workload in under 15 minutes.

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| [Go](https://go.dev/doc/install) | 1.25.5+ | Building Propeller binaries |
| [Rust](https://rustup.rs/) | Latest | Building the proplet runtime |
| [Docker](https://docs.docker.com/) | 20.10+ | Running SuperMQ services |
| [Make](https://www.gnu.org/software/make/) | 3.81+ | Build automation |

:::note Optional Tools
These are only needed if you plan to build WASM modules locally:
- [TinyGo](https://tinygo.org/getting-started/install/) (0.34.0+) - Compile Go to WASM
- [Wasmtime](https://wasmtime.dev/) - Local WASM testing
:::

## Quick Start (5 minutes)

### Step 1: Clone and Build

```bash
git clone https://github.com/absmach/propeller.git
cd propeller
make all -j $(nproc)
make install
```

Expected output:
```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build ... -o build/manager cmd/manager/main.go
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build ... -o build/cli cmd/cli/main.go
cd proplet && cargo build --release && cp target/release/proplet ../build
    Finished `release` profile [optimized] target(s) in 15.36s
```

:::tip Build Issues?
See the [Developer Guide](./developer-guide#troubleshooting) for common build problems.
:::

### Step 2: Start SuperMQ

SuperMQ provides the messaging infrastructure for Propeller.

```bash
make start-supermq
```

Wait ~30 seconds for all services to initialize. Verify with:

```bash
docker ps | grep supermq
```

### Step 3: Provision Propeller

Create the required domain, clients, and channels:

```bash
propeller-cli provision
```

Follow the interactive prompts. On success:
```bash
Successfully created config.toml file
```

### Step 4: Configure and Restart

Copy the configuration to Docker:

```bash
cp config.toml docker/config.toml
```

Enable the config mount in `docker/compose.yaml` by uncommenting the volumes for manager, proplet, and proxy services. Then restart:

```bash
make stop-supermq
make start-supermq
```

### Step 5: Verify Deployment

Check that all services are running:

```bash
# Manager logs
docker logs propeller-manager | tail -5
```

Expected:
```json
{"level":"INFO","msg":"MQTT connection established"}
{"level":"INFO","msg":"manager service http server listening at manager:7070 without TLS"}
```

```bash
# Proplet logs
docker logs propeller-proplet | tail -5
```

Expected:
```
INFO Starting Proplet (Rust) - Instance ID: ...
INFO Published discovery message
INFO Subscribed to topic: .../control/manager/start
```

## Deploy Your First Workload

### List Available Proplets

```bash
curl http://localhost:7070/proplets | jq
```

Response:
```json
{
  "total": 1,
  "proplets": [
    {
      "id": "a95517f9-5655-4cf5-a7c8-aa00290b3895",
      "name": "Dankert-Gene",
      "alive": true
    }
  ]
}
```

### Create a Task

```bash
curl -X POST "http://localhost:7070/tasks" \
  -H "Content-Type: application/json" \
  -d '{"name": "add", "inputs": [10, 20]}'
```

Save the returned `id` for the next steps.

### Upload WASM Module

```bash
curl -X PUT "http://localhost:7070/tasks/<task_id>/upload" \
  -F "file=@build/addition.wasm"
```

### Start the Task

```bash
curl -X POST "http://localhost:7070/tasks/<task_id>/start"
```

Response:
```json
{"started": true}
```

### Check Results

```bash
curl "http://localhost:7070/tasks/<task_id>" | jq
```

The task `state` will change from `0` (pending) to `2` (completed) with the result in the `output` field.

## Alternative: Run Without Docker

If you prefer running Propeller binaries directly (useful for development):

```bash
# Stop Docker-based Propeller services
docker stop propeller-manager propeller-proxy propeller-proplet

# Start services locally (each in a separate terminal)
propeller-manager
propeller-proplet
propeller-proxy
```

Both methods are functionally equivalent. Docker is recommended for production, while local binaries are easier for debugging.

## Troubleshooting

### `make install` fails with permission denied

Your `$GOBIN` path isn't configured:

```bash
export GOBIN=$HOME/go/bin
export PATH=$PATH:$GOBIN
make install
```

### `propeller-cli provision` fails

Ensure SuperMQ is fully started:

```bash
docker logs supermq-users | tail -10
```

Look for: `users service http server listening at users:9002`

### Services can't find config.toml

The config must be mounted into Docker containers. Verify:

```bash
ls docker/config.toml
```

If missing, copy it: `cp config.toml docker/config.toml`

### WASM execution fails

Check proplet logs for runtime errors:

```bash
docker logs propeller-proplet | grep -i error
```

For detailed troubleshooting, see the [Developer Guide](./developer-guide#troubleshooting).

## Stopping Propeller

```bash
make stop-supermq
```

This stops all SuperMQ and Propeller containers while preserving data volumes.
