---
title: Proplet
description: Worker nodes that execute WebAssembly tasks in the Propeller system
---

The `proplet` is a Rust-based worker node that executes WebAssembly workloads on behalf of the Manager. It connects to the Manager over MQTT, receives task start and stop commands, fetches WebAssembly binaries from the Proxy in chunks, executes them using an embedded or external Wasmtime runtime, and publishes results back over MQTT.

## Runtimes

The proplet supports three execution modes, selected automatically based on configuration:

| Mode | Description |
| ---- | ----------- |
| **Embedded Wasmtime** | Default. Executes Wasm in-process using an embedded Wasmtime instance. No external binary required. |
| **Host Runtime** | Set `PROPLET_EXTERNAL_WASM_RUNTIME` to the path of an external Wasm runtime executable (e.g. `/usr/bin/wasmtime`). The proplet invokes the binary as a subprocess, passing the Wasm file and any `cli_args` from the task. |
| **TEE Runtime** | Activated automatically when a Trusted Execution Environment is detected at startup. Decrypts encrypted OCI-packaged Wasm images using the Attestation Agent and KBS before executing them in the protected environment. |

## Configuration

The proplet is configured using environment variables. Values can also be loaded from a `config.toml` file (used by the Docker Compose stack).

### Core

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_LOG_LEVEL` | Log level (`debug`, `info`, `warn`, `error`) | `info` |
| `PROPLET_INSTANCE_ID` | Unique identifier for this proplet instance | Auto-generated UUID |
| `PROPLET_LIVELINESS_INTERVAL` | Heartbeat publish interval in seconds | `10` |
| `PROPLET_METRICS_INTERVAL` | Proplet-level metrics publish interval in seconds | `10` |
| `PROPLET_ENABLE_MONITORING` | Enable per-task process monitoring | `true` |
| `PROPLET_PULL_CONCURRENT_LIMIT` | Maximum concurrent OCI layer pull operations | `4` |
| `PROPLET_LAYER_STORE_PATH` | Directory for OCI layer assembly | `/tmp/proplet/layers` |

### MQTT / SuperMQ

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_MQTT_ADDRESS` | MQTT broker URL | `tcp://localhost:1883` |
| `PROPLET_MQTT_TIMEOUT` | MQTT operation timeout in seconds | `30` |
| `PROPLET_MQTT_QOS` | MQTT Quality of Service level | `2` |
| `PROPLET_MQTT_KEEP_ALIVE` | MQTT keepalive interval in seconds | `30` |
| `PROPLET_MQTT_MAX_PACKET_SIZE` | Maximum MQTT packet size in bytes | `10485760` |
| `PROPLET_MQTT_INFLIGHT` | Maximum in-flight MQTT messages | `10` |
| `PROPLET_DOMAIN_ID` | SuperMQ domain identifier | _(required)_ |
| `PROPLET_CHANNEL_ID` | SuperMQ channel identifier | _(required)_ |
| `PROPLET_CLIENT_ID` | SuperMQ client identifier for MQTT authentication | _(required)_ |
| `PROPLET_CLIENT_KEY` | SuperMQ client key for MQTT authentication | _(required)_ |

### Runtime

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_EXTERNAL_WASM_RUNTIME` | Path to an external Wasm runtime executable. If empty, uses the embedded Wasmtime runtime. | `""` |
| `PROPLET_DIRS` | Colon-separated list of host directories to preopen for WASI access (e.g. `/data:/tmp`). | `""` |

### HTTP Proxy (optional)

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_HTTP_ENABLED` | Enable the built-in HTTP proxy server | `false` |
| `PROPLET_HTTP_PROXY_PORT` | Port for the built-in HTTP proxy server | `8222` |

### HAL (Hardware Abstraction Layer)

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_HAL_ENABLED` | Enable the Hardware Abstraction Layer (used for TEE attestation helpers) | `true` |

### Kubernetes

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_MANAGER_K8S_NAMESPACE` | Kubernetes namespace override for K8s-based deployments | `""` |

### MQTT Advanced

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_MQTT_REQUEST_CHANNEL_CAPACITY` | Internal channel buffer size for outgoing MQTT requests | `128` |

### Config File Fallback

If `PROPLET_DOMAIN_ID`, `PROPLET_CLIENT_ID`, `PROPLET_CLIENT_KEY`, or `PROPLET_CHANNEL_ID` are not set via environment variables, the proplet attempts to read them from a TOML config file.

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_CONFIG_FILE` | Path to the TOML config file | `config.toml` |
| `PROPLET_CONFIG_SECTION` | Section name within the TOML config file | `proplet` |

### TEE (Trusted Execution Environment)

These variables are only required when running inside a TEE and processing encrypted workloads.

| Variable | Description | Default |
| -------- | ----------- | ------- |
| `PROPLET_KBS_URI` | URI of the Key Broker Service (e.g. `http://10.0.2.2:8082`) | `""` |
| `PROPLET_AA_CONFIG_PATH` | Path to the Attestation Agent configuration file | `""` |

> **Important:** If a TEE is detected at startup and `PROPLET_KBS_URI` is not set, the proplet will exit with an error. `PROPLET_KBS_URI` is mandatory whenever a TEE is present.

## Usage

### Using the Embedded Wasmtime Runtime

By default the proplet uses its built-in Wasmtime instance. Set the required environment variables and start:

```bash
export PROPLET_DOMAIN_ID="your_domain_id"
export PROPLET_CHANNEL_ID="your_channel_id"
export PROPLET_CLIENT_ID="your_client_id"
export PROPLET_CLIENT_KEY="your_client_key"
propeller-proplet
```

Startup logs confirm the runtime in use:

```bash
INFO Starting Proplet (Rust) - Instance ID: 67b2bcb2-9c56-4c57-b163-085d6ec2c313
INFO MQTT client created (TLS: false)
INFO Using Wasmtime runtime
INFO Starting MQTT event loop
INFO Starting PropletService
INFO Published discovery message
INFO Subscribed to topic: m/.../control/manager/start
INFO Subscribed to topic: m/.../control/manager/stop
INFO Subscribed to topic: m/.../registry/server
```

### Using a Host WebAssembly Runtime

To use an external runtime binary such as `wasmtime` or `wasmer`, set `PROPLET_EXTERNAL_WASM_RUNTIME`:

```bash
export PROPLET_DOMAIN_ID="your_domain_id"
export PROPLET_CHANNEL_ID="your_channel_id"
export PROPLET_CLIENT_ID="your_client_id"
export PROPLET_CLIENT_KEY="your_client_key"
export PROPLET_EXTERNAL_WASM_RUNTIME="/usr/bin/wasmtime"
propeller-proplet
```

The proplet will invoke the specified binary as a subprocess. You will need to supply any required runtime flags via the task's `cli_args` field. For example, to run the `addition` example with `wasmtime --invoke add`:

```json
{
  "name": "add",
  "cli_args": ["--invoke", "add"],
  "inputs": [10, 20]
}
```

### Using TEE Mode

The proplet automatically detects the Trusted Execution Environment at startup by inspecting device files and CPU information. If a TEE is detected and `PROPLET_KBS_URI` is set, the TEE runtime is activated for tasks marked `encrypted: true`.

Supported TEE types detected at startup:

| TEE | Detection method |
| --- | ---------------- |
| Intel TDX | `/dev/tdx_guest`, `/sys/firmware/tdx_guest`, cpuinfo/dmesg TDX flags |
| AMD SEV/SNP | `/dev/sev`, EFI variables (`SevStatus`), cpuinfo/dmesg SEV flags |
| Intel SGX | `/dev/sgx_enclave`, `/dev/sgx/enclave`, `/dev/isgx` (legacy), cpuinfo/dmesg SGX flags |

Set the TEE environment variables before starting the proplet:

```bash
export PROPLET_DOMAIN_ID="your_domain_id"
export PROPLET_CHANNEL_ID="your_channel_id"
export PROPLET_CLIENT_ID="your_client_id"
export PROPLET_CLIENT_KEY="your_client_key"
export PROPLET_KBS_URI="http://10.0.2.2:8082"
export PROPLET_AA_CONFIG_PATH="/etc/default/proplet.toml"
propeller-proplet
```

Startup logs show the detected TEE type:

```bash
Intel TDX Detection:
  - Intel CPU: true
  - /dev/tdx_guest: true
  - TDX CPU flag: true
INFO TEE runtime initialized successfully
```

See the [Encrypted workloads](tee) guide for the full setup flow.

## Task Start Flow

1. The Manager publishes a start command to `m/{domain_id}/c/{channel_id}/control/manager/start` with the task payload.
2. If the task has an `image_url`, the proplet publishes a fetch request to `m/{domain_id}/c/{channel_id}/registry/proplet`.
3. The Proxy streams Wasm binary chunks to `m/{domain_id}/c/{channel_id}/registry/server`. The proplet assembles the chunks (incomplete assemblies expire after a 5-minute TTL).
4. If the task is `encrypted: true`, the TEE runtime decrypts the binary using the key fetched from KBS via the Attestation Agent.
5. The proplet executes the Wasm binary using the configured runtime.
6. Results are published to `m/{domain_id}/c/{channel_id}/control/proplet/results`.

## Task Stop Flow

The Manager publishes a stop command to `m/{domain_id}/c/{channel_id}/control/manager/stop` with the task ID. The proplet terminates the running task and releases its resources.

## Liveliness and Metrics

The proplet sends two types of periodic MQTT messages:

| Topic | Content | Default interval |
| ----- | ------- | ---------------- |
| `m/{domain_id}/c/{channel_id}/control/proplet/alive` | Heartbeat (proplet ID + channel ID) | 10 s (`PROPLET_LIVELINESS_INTERVAL`) |
| `m/{domain_id}/c/{channel_id}/control/proplet/metrics` | Proplet-level CPU, memory, and cgroup stats | 10 s (`PROPLET_METRICS_INTERVAL`) |

When monitoring is enabled, per-task metrics are published while the task is running:

| Topic | Content |
| ----- | ------- |
| `m/{domain_id}/c/{channel_id}/control/proplet/task_metrics` | CPU %, memory, disk I/O, thread count, file descriptors |

## Discovery

On startup the proplet publishes a discovery message to:

```
m/{domain_id}/c/{channel_id}/control/proplet/create
```

The Manager records the proplet's ID and channel from this message. If the proplet disconnects unexpectedly, the MQTT broker delivers the Last Will message on the `control/proplet/alive` topic, notifying the Manager.

## MQTT Topic Reference

| Topic | Direction | Description |
| ----- | --------- | ----------- |
| `m/{domain_id}/c/{channel_id}/control/proplet/create` | Proplet → Manager | Startup discovery announcement |
| `m/{domain_id}/c/{channel_id}/control/proplet/alive` | Proplet → Manager | Periodic heartbeat |
| `m/{domain_id}/c/{channel_id}/control/proplet/metrics` | Proplet → Manager | Proplet-level resource metrics |
| `m/{domain_id}/c/{channel_id}/control/proplet/task_metrics` | Proplet → Manager | Per-task process metrics |
| `m/{domain_id}/c/{channel_id}/control/proplet/results` | Proplet → Manager | Task execution results |
| `m/{domain_id}/c/{channel_id}/control/manager/start` | Manager → Proplet | Task start command |
| `m/{domain_id}/c/{channel_id}/control/manager/stop` | Manager → Proplet | Task stop command |
| `m/{domain_id}/c/{channel_id}/registry/proplet` | Proplet → Proxy | Request Wasm binary chunks |
| `m/{domain_id}/c/{channel_id}/registry/server` | Proxy → Proplet | Wasm binary chunk delivery |
